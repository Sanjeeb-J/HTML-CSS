Chrome DevTools

1. What is Chrome DevTools?

Chrome DevTools is a set of developer tools built into Google Chrome that helps developers understand, debug, and improve websites.

You can use it to:

  • Inspect HTML & CSS (see structure and design)
  • Debug JavaScript (fix logic errors)
  • Analyze performance (find slow parts)
  • Monitor network requests (APIs, files)
  • Test responsive design (mobile view)
“DevTools lets you see what is happening behind the website.”

How to Open DevTools:

  • Right click → Inspect
  • Ctrl + Shift + I (Windows/Linux)
  • Cmd + Option + I (Mac)
  • F12
You can open DevTools on any website, not just your own.

2. DevTools Interface Overview

When DevTools opens, it can look complex. Here is the layout clearly explained:

Main Parts:

  • Panel Tabs → Elements, Console, Network, etc.
  • Toolbar → Controls and settings
  • Main Panel Area → Where you work
  • Drawer (bottom panel) → Extra tools
“Each tab solves a different type of problem.”

3. Elements Panel

Purpose:

Used to inspect and edit HTML & CSS in real-time. This is the most used panel for beginners.

  • Inspect Element: Click the inspect icon, hover over elements to highlight them, and click to select them in the HTML.
  • Edit HTML: Double click text to change content, or right-click → Edit as HTML.
  • Styles Pane: Shows all CSS applied to the selected element. You can add new styles, disable them (uncheck), or modify values.
  • Box Model: Shows Content, Padding, Border, and Margin to help understand spacing.
  • Computed Styles: Shows final styles after all CSS is applied.
  • Responsive Mode: Toggle the device toolbar to select mobile devices and change screen dimensions.

4. Console Panel

Purpose:

Used to run JavaScript and see errors.

console.log("Hello")
console.error("Error")
console.warn("Warning")
  • Inspect Variables: Test code directly in the browser.
  • Select Elements: document.querySelector("h1")
  • Shortcuts: $0 for current element, clear() to clear console.
  • YouTube Hack: document.querySelector('video').playbackRate=3;

5. Sources Panel (JavaScript Debugging)

Purpose:

Used for debugging JavaScript step-by-step.

  • File Navigation: See all JS files loaded on the page.
  • Breakpoints: Click line number to pause execution.
  • Step Controls: Step over, Step into, and Resume.
  • Watch Variables: Track values in real-time.
  • Call Stack: See function execution flow.

6. Network Panel

Purpose:

Shows all network requests made by the page.

  • Reload Page: You must reload to see the initial requests.
  • Types: Fetch/XHR, JS, CSS, Images, etc.
  • Status Codes: 200 (Success), 404 (Not Found), 500 (Server Error).
  • Throttling: Simulate slow internet.

7. Application Panel

Purpose:

View and manage data stored in the browser (Local Storage, Session Storage, Cookies).

8. Performance & Lighthouse

  • Performance: Analyze website speed, FPS, and CPU usage.
  • Lighthouse: Audit performance, SEO, accessibility, and best practices.
